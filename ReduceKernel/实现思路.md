# Reduction实现

本仓库实现了 CUDA 经典的并行求和规约（Sum Reduction）。重点展示了如何通过减少 Kernel 启动开销、提升指令级并行度（ILP）以及优化全局访存带宽，实现接近硬件极限的吞吐量。

---

### 1. Reduce Naive - 经典两阶段规约
**实现逻辑**：采用经典的树状规约架构。由于 CUDA Block 间无法直接同步，该版本通过两次 Kernel Launch 完成：第一次将大数组规约至中间结果，第二次将中间结果规约至最终标量。

#### 思路
* **Warp Shuffle Primitives**：摒弃了传统的基于 Shared Memory 的规约，直接使用 `__shfl_down_sync` 在寄存器层面进行数据交换。寄存器是最快的，也不用考虑共享内存上的bank冲突。
* **分级规约架构**：先进行 Warp 内规约，再通过 Shared Memory 汇总各 Warp 结果，最后由第一个 Warp 完成 Block 级的最终汇总。逻辑严密，通过 `__syncthreads()` 确保了内存可见性。

---

### 2. Reduce Optimized - 向量化与单阶段原子规约
**实现逻辑**：在 Naive 的基础上引入了 **float4 向量化读取** 和 **Atomic Aggregation**，将性能推向带宽受限（Memory-bound）的顶峰。

#### 思路
* **LDG.128 向量化加载 (Vectorized Access)**：
    * 通过 `reinterpret_cast<const float4*>`，使每个线程单次指令即可读取 128-bit 数据（4 个 float）。
* **双预取 (Double Prefetching)**：
    * 相比 Naive 版本每个线程处理 2 个元素，优化版每个线程通过 `float4` 结合手动解构，处理了 8 个元素（2 个 float4）。
    * 此举缓解了规约时，后半部分线程发呆的局面。
