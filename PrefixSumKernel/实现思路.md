# GPU 并行前缀和（Prefix Sum）技术实践总结

本项目通过五种不同级别的 CUDA Kernel 实现，深入探讨了并行计算中的经典优化手段，涵盖了从基础算法到硬件级原语的应用。

---

## 1. 核心算法实现方案对比

### A. Naive Scan (基础迭代)
- **原理**：基于 Hillis-Steele 算法，每轮迭代让线程 `i` 读取 `i-s` 的值并累加。
- **技术点**：
    - **双重同步 (Double Sync)**：在共享内存（Shared Memory）中通过两次 `__syncthreads()` 解决“读后写”（RAW）冲突，确保读取的是上一轮的旧值。
- **评价**：代码最简洁，但在小规模数据下由于逻辑简单，性能表现意外地稳健。

### B. Double Buffer Scan (双缓冲优化)
- **原理**：申请两块共享内存空间（T1, T2），一读一写。
- **技术点**：
    - **指针交换 (Pointer Swap)**：每轮迭代后交换读写指针，消除了 Naive 版本中第二次同步的必要性。
    - **最终指向**：注意迭代次数为偶数还是奇数，确保从正确的缓冲区写回全局内存。
- **评价**：有效减少了同步开销，是处理共享内存竞争的标准做法。

### C. Brent-Kung Tree (平衡树算法)
- **原理**：分为 **Sweep Up**（规约阶段）和 **Sweep Down**（分发阶段）。
- **技术点**：
    - **计算密度**：加法次数从 $O(n \log n)$ 降低到 $O(n)$。
    - **索引映射**：通过复杂的位移运算定位左右子节点，对线程进行非连续映射。
- **评价**：在大规模数据下理论效率最高，但由于索引计算开销大，在 $N=1024$ 时优势不明显。

### D. Bank Conflict Free (规约避冲)
- **原理**：针对共享内存的硬件特性（32个 Bank）进行布局优化。
- **技术点**：
    - **Padding (填充)**：在每 32 个元素后插入一个空位 `shared_input[n + (n >> 5)]`。
    - **逻辑变换**：利用 `OFFSET(n)` 宏将逻辑索引映射到物理地址，使原本会撞在同一个 Bank 的 2 的幂次步长访问强制错开。
- **评价**：解决了 Shared Memory 的串行化访问问题，是高性能算子的必修课。

### E. Warp Shuffle Hierarchical (寄存器级优化)
- **原理**：利用 NVIDIA 硬件原语 `__shfl_up_sync` 在 Warp 内部直接交换寄存器数据。
- **技术点**：
    - **无内存访问**：跳过共享内存，直接在寄存器层面完成 32 线程的扫描，延迟极低。
    - **层次化结构**：Warp 内 Scan -> 共享内存存储 Warp 总和 -> Warp 0 再次 Scan -> 补偿偏移。
- **评价**：单 Block 性能的天花板，避开了几乎所有内存相关的瓶颈。

## 2. 性能优化阶梯
1. **第一阶段**：实现算法，保证结果正确（同步与边界处理）。
2. **第二阶段**：减少同步频率，利用双缓冲或寄存器通信。
3. **第三阶段**：处理硬件冲突（Bank Conflict）和分支发散。
4. **第四阶段**：精简索引计算，使用宏或内联函数减少冗余指令。
