# Matrix Transpose

本仓库实现了多种 CUDA 矩阵转置算法，核心目标是解决转置操作中固有的“非连续访存”难题。通过引入 Shared Memory 作为中间跳板，将全局内存的非合并写操作转化为高效的合并访存，并针对硬件 Bank 结构进行了深度去冲突优化。

---

### 核心挑战：非合并访存 (Uncoalesced Access)
在 Naive 实现中，如果读取是横向连续的，则写入必然是纵向非连续的。由于 GPU 显存总线以 32-byte 或 128-byte 粒度触发，非连续写入会导致巨大的带宽浪费。

---

### 技术挖掘点 (Interview-Ready)

* **Tiled Shared Memory 缓存策略**：
    * **实现细节**：利用 `__shared__ float cache[SM][SN]` 划分子块（Tile）。线程块先将全局内存中的行连续读入共享内存，同步后再按列顺序写回全局内存。

* **Padding 技巧消除 Bank Conflict (Conflict Free +1)**：
    * **实现细节**：在声明共享内存时使用 `[SM][SN + 1]`，经典的Bank冲突消除技巧。
    
* **线性索引重映射 (Index Remapping)**：
    * **实现细节**：在 `transpose_shared_direct_map` 中，通过手动计算 `block_linear_index` 并重新推导 `shared_offset_row/col`。

* **双路预取与指令并行 (Double Fetch / Vectorized)**：
    * **实现细节**：在 `double_fetch` 版本中，每个线程负责读取两个元素，减少了一半的 Block 数量。
    * 通过在单个循环内发射更多的访存指令，可以更好地掩盖显存访问延迟（Latency Hiding），并减少线程发射（Launch）和同步的硬件开销。
