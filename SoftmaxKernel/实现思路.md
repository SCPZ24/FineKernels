# Softmax Kernel实现思路

本仓库包含两个版本的 CUDA Softmax 实现。重点展示了如何通过利用显存架构特性与 Warp-level Primitives，将计算延迟从访存受限（Memory-bound）逐步推向硬件极限。

---

### 1. Softmax Forward (Naive) - 基础块级并行
**实现逻辑**：每个 Block 处理一行（Row）数据，利用 Shared Memory 进行块内规约。

#### 思路
* **两级规约架构**：实现了 `warpReduceMax` 和 `warpReduceSum`。首先在 Warp 内部使用 `__shfl_xor_sync`（纠删码同步）进行高效交换，减少了对 Shared Memory 的依赖。
* **抗溢出设计**：遵循标准 Safe-Softmax 流程，先提取 $max(x)$ 并在指数运算中减去，确保了在 FP32 数值范围内的计算稳定性。
* **动态共享内存控制**：通过 `extern __shared__` 动态分配空间存储每个 Warp 的中间结果，适应不同线程配置，保证了算法的泛化能力。

---

### 2. Softmax Forward (Optimized) - 性能优化
**实现逻辑**：针对特定通道数（$C$），采用 **Warp-Per-Row** 策略，结合向量化读写与寄存器缓存。

#### 思路
* **向量化访存 (Vectorized Memory Access)**：
    * 通过 `float4` 强制类型转换，将原本的 4 次 32 位读请求合并为 1 次 128 位指令。
    * 这极大地提升了总线利用率，在相同时间内搬运更多数据，减少了 Global Memory 的指令发射开销。
* **寄存器压力与计算掩盖 (Register Tiling)**：
    * 引入 `reg_val_f4` 寄存器数组，将整行数据在第一次循环读取后**常驻寄存器**。
    * 规避反复读取 Global Memory，（实现了 **Zero-Redundant-Read**）。由于 $C$ 较小，数据完全在寄存器中存取，访存开销被完美隐藏。
* **规避共享内存瓶颈**：
    * 由于采用 Warp-Per-Row 架构，所有规约操作均在 Warp 内部通过 `__shfl_sync` 原语完成。
    * 去除了 `__syncthreads()` 同步指令和 Shared Memory 访存。这不仅消除了 Bank Conflict 的可能性，还降低了流水线停顿，显著提升了指令吞吐量（Throughput）。
