# RMSNorm (Root Mean Square Layer Normalization)

本仓库展示了 LLM 推理流水线中核心算子 RMSNorm 的两种优化路径。通过向量化访存（Vectorized Access）与任务流坍缩（Task Collapse）技术，显著降低了该 Memory-bound 算子的访存延迟，并优化了在不同 Batch Size 下的硬件利用率。

---

### 🛠️ 核心思路：从逐行处理到网格坍缩
RMSNorm 的性能瓶颈在于计算每一行的平方和规约。本实现对比了“一线程块处理一行”与“一线程块多行并发”两种策略，在确保数值稳定性的同时，追求极高的有效带宽。

---

### 思路

用float4总归能让总线程数从 N * C 变为 N * C / 4。
有两种方法：
1. Block数不变，一个block里的线程数变为原来的1/4。
2. 一个block里的线程数不变，但是每个block处理4行数据。block数变为1/4.

* **Block Collapse (线程块内任务优化)**：
    * **实现细节**：在 `RMSNorm_block_collapse` 中，一个 Block 负责一行规约。利用 `warpReduceSum` 在寄存器层面通过 `__shfl_down_sync` 完成求和。

* **Grid Collapse (网格级任务重组 - 核心亮点)**：
    * **实现细节**：在 `RMSNorm_grid_collapse` 中，一个 Block 同时处理 4 行数据，并使用 `__shared__ float shared_wrap_data[4][9]` 并行管理各行的中间状态。

综合来看，第二种方法更优。
